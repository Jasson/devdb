如何使用github提供的服务?
1. git config --global user.name erlangonrails
   erlangonrails是你的名字

2. git config --global user.email erlangonrails@gmail.com

3. 产生一个SSH key pair:
   a. 检查ssh key directory是否已经存在
      cd ~/.ssh
      ls ~/.ssh
      known_hosts
   b. 使用如下命令产生一个key (所有需要输入的地方都使用回车, 使用默认设置)
      ssh-keygen -t rsa -C "erlangonrails@gmail.com"
      Generating public/private rsa key pair.
      Enter file in which to save the key (/home/liqiang/.ssh/id_rsa): 
      Enter passphrase (empty for no passphrase): 
      Enter same passphrase again: 
      Your identification has been saved in /home/liqiang/.ssh/id_rsa.
      Your public key has been saved in /home/liqiang/.ssh/id_rsa.pub.
      The key fingerprint is:
      27:07:60:d2:82:5e:24:39:2d:2f:5c:6a:69:bd:df:9e erlangonrails@gmail.com

   c. ls ~/.ssh 
      id_rsa  id_rsa.pub  known_hosts
   d. 拷贝id_rsa.pub中所有的内容到https://github.com/account
      的SSH Public Keys Tab, 点击Add another public key, 然后把内容复制进去.
      标题默认为空即可, 会自动设置为账户的email地址.

4. Next steps:
   a. cd ~/github
   b. mkdir devdb
   c. cd devdb
   d. git init
      Initialized empty Git repository in /home/liqiang/github/devdb/.git/
   e. touch README
   f. git add README
   g. git commit -m 'add README file'
      [master (root-commit) d698d3d] add README file
      1 files changed, 1 insertions(+), 0 deletions(-)
      create mode 100644 README
   h. git remote add origin git@github.com:erlangonrails/devdb.git
      定义远程服务器别名origin
   i. git push origin master
      (本地和远程合并, 本地默认分支为master)更新到远程服务器
      The authenticity of host 'github.com (207.97.227.239)' can't be established.
      RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.

      Warning: Permanently added 'github.com,207.97.227.239' (RSA) to the list of known hosts.
      Counting objects: 3, done.
      Writing objects: 100% (3/3), 265 bytes, done.
      Total 3 (delta 0), reused 0 (delta 0)
      To git@github.com:erlangonrails/devdb.git
      * [new branch]      master -> master

4. github签入文件的流程就是add, commit, push
   git add newfile
   git commit -m 'this is the checkin comments'
   git push origin master
